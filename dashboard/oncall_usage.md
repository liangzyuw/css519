# On-call usage for CoffeeJelly Dashboard

## Service Health:
Identifies whether or not the backend service and frontend are available to users and such.

## API Metrics: 
Displays how many requests are made by students and instructors on the platform, and how many are successful. Significant amounts of unsuccessful requests would indicate possible issues that majority of users are facing. 

## Users:
Shows how many active users there are currently, and how many annotations are made total across all textbooks on the platform. If this number reaches 0, then it's possible that authentication is having issues. 

## Security:
Identifies any unauthorized requests recently made. If any do show up, it indicates a possible breach of access. 

## Deployment: 
Displays current version that is deployed to the current user base.
